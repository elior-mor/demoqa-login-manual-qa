## QA Baseline Project

### Project Overview
This project is a baseline QA portfolio project designed to demonstrate a structured and professional manual testing process.

The testing was conducted on the **Login page** of the **DemoQA web application**, covering functional validation, negative scenarios, usability and basic accessibility checks across desktop and mobile platforms.

---

### Project Objectives
- Demonstrate a complete manual QA workflow
- Apply structured test planning and execution
- Document test cases and defects in a professional manner
- Present clear and concise QA documentation suitable for portfolio review

---

### Application Under Test
- **Application:** DemoQA Web Application  
- **Module:** Login Page  
- **URL:** https://demoqa.com/login  
- **Platforms Tested:**  
  - Desktop (Windows 11, Google Chrome)  
  - Mobile (Android, Google Chrome)

---

### Test Artifacts
The following documents are included in this repository:

- **Software Test Plan (STP)**  
  Defines the testing scope, strategy, environments, risks and tools.

- **Test Cases Document**  
  Contains 10 structured test cases covering positive, negative, validation and navigation scenarios.

- **Bug Reports**  
  Includes 2 documented defects related to usability, accessibility and validation behavior.

- **Software Test Report (STR)**  
  Summarizes the test execution results, findings and final conclusions.

---

### Test Summary
- **Total Test Cases Executed:** 10  
- **Passed:** 10  
- **Failed:** 0  
- **Total Defects Found:** 2  
  - Severity: Minor  
  - Categories: Usability / Accessibility / Validation

---

### Key Findings
- Validation feedback for required fields behaves inconsistently between desktop and mobile platforms.
- Mobile users do not receive explanatory feedback when interacting with mandatory field indicators.
- Error messages for invalid login attempts are generic and do not differentiate between input issues.
- Core login functionality behaves as expected.
